# Data Model: Environment Lookup Function

## Entity: LookupRequest

Represents the caller input used to query one environment variable.

- Fields:
  - `name` (string, required): raw variable name input from caller.
- Validation rules:
  - No trimming.
  - No additional format validation.
  - Empty string is accepted and passed through unchanged.
- Relationships:
  - Produces exactly one `LookupOutcome`.

## Entity: LookupOutcome

Represents either a resolved value or a missing-variable error for a `LookupRequest`.

- Fields:
  - `found` (bool): result from `os.LookupEnv`.
  - `value` (string): resolved value when `found=true`; may be empty string.
  - `id` (string): canonical ID for state-backed methods; set equal to request `name`.
- Validation rules:
  - If `found=true`, success path is used even when `value==""`.
  - If `found=false`, emit canonical missing-variable error message.
- Relationships:
  - `LookupOutcome` is generated by `LookupBehaviorRule` and consumed by all lookup methods.

## Entity: LookupBehaviorRule

Canonical shared behavior contract implemented once and reused by all surfaces.

- Responsibilities:
  - Accept raw name input.
  - Execute `os.LookupEnv(name)`.
  - Map lookup result into success/error outcome.
  - Emit one shared, byte-for-byte identical user-facing missing-variable message.
- Relationships:
  - Invoked by non-sensitive data source read path.
  - Invoked by sensitive data source indirectly through non-sensitive data source read path.
  - Invoked by provider-defined lookup function.
  - Designed for direct reuse by a future resource read path.

## Entity: SensitiveHandlingGuidance

Documentation-level policy for secret-related use cases.

- Fields:
  - `function_support` (enum): `non-sensitive-only`.
  - `recommended_sensitive_path` (string): `environment_sensitive_variable` data source.
  - `terraform_expression_guidance` (string): use Terraform `sensitive(...)` handling where needed.
- Validation rules:
  - Must explicitly state no separate sensitive function is provided.
  - Must differentiate function output handling from sensitive data source semantics.

## State Transitions

1. `LookupRequest` received with raw `name`.
2. Shared `LookupBehaviorRule` evaluates `os.LookupEnv(name)`.
3. If found:
   - emit success `LookupOutcome` (`value`, optional `id=name` for state-backed methods).
4. If missing:
   - emit canonical missing-variable error.
5. For documentation flows:
   - route users with sensitive needs to `SensitiveHandlingGuidance` recommendations.
