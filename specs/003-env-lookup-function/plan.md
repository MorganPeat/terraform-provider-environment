# Implementation Plan: Environment Lookup Function

**Branch**: `003-env-lookup-function` | **Date**: 2026-03-24 | **Spec**: `/workspaces/terraform-provider-environment/specs/003-env-lookup-function/spec.md`  
**Input**: Feature specification from `/workspaces/terraform-provider-environment/specs/003-env-lookup-function/spec.md`

## Summary

Add one provider-defined function for environment variable lookup and centralize lookup behavior in one shared helper reused by all in-scope lookup surfaces (non-sensitive data source, sensitive data source, function). Preserve current raw-name semantics (no trimming/extra validation), keep missing-variable errors byte-for-byte identical across methods, and explicitly avoid adding a separate sensitive function. Document sensitive handling guidance and add parity-focused acceptance-style tests.

## Technical Context

**Language/Version**: Go 1.25.0  
**Primary Dependencies**: `terraform-plugin-framework` v1.19.0, `terraform-plugin-testing` v1.15.0, `terraform-plugin-docs` v0.24.0  
**Storage**: N/A (reads process environment via `os.LookupEnv`; values flow into Terraform state where applicable)  
**Testing**: `go test` via `make test`; acceptance-style provider tests (`TestAcc...` with `IsUnitTest: true`)  
**Target Platform**: Terraform CLI runtime on Linux/macOS/Windows where environment variables are available  
**Project Type**: Terraform provider plugin (single Go module)  
**Performance Goals**: Keep lookup path constant-time and local-only; no additional I/O or network calls  
**Constraints**: Byte-for-byte identical missing-variable message across non-sensitive data source/sensitive data source/function; no name normalization; no separate sensitive function  
**Scale/Scope**: One shared lookup helper, one new provider-defined function surface, parity tests across all in-scope methods, and documentation/example updates
